Finmondo

The next generation Irish Challenger Bank for Start-ups and Micro-Enterprises.

I was hired by Finmondo as a UI Designer to create a proof-of-concept web platform, which solved the issues start-ups face when trying to open a business bank account.

The deliverable for this project is a fully-interactive prototype that can be taken to stakeholders and passed on to developers. This project is still a work in progress!

Tools used: Adobe XD, Adobe Illustrator.

What is Finmondo?

Finmondo aims to be the next generation Irish Challenger Bank for Startups and Micro-Enterprises.

When opening business bank accounts, entrepreneurs, startups and small businesses are plagued with complicated paperwork, limited offerings, a lack of communication throughout the process, and a very long process. Finmondo's missions is to "build a fairer and more efficient Business Bank that works for the customer, not against them.

Finmondo contacted me to help redesign their web platform, as their current product design didn't match their brand. I was provided with a glimpse of their current product and some wireframes to create a platform that was professional, clean and, above all, easy to use.

UI Design

The main issues the team at Finmondo had with their existing platform design was that they felt it looked too cluttered visually, and that it didn't match their branding which makes use of bright colours, and the navigation felt overly complicated.

From the outset I made sure my design made use of a feature colour on interactive elements or where we need to draw the users' attention. It was important to use plent of whitespace in the UI centre stage to prevent the use of colour from becoming overwhelming, as well as the sheer volume of data that can be present on some screens.

I decided to use a fixed left navigation bar so it was easily accessible, and it meant that we could use disclosures if necessary, but it wouldn't block the UI in any way.

Project Reflection

This project is still ongoing, but I am learning a lot about working with a startup and how to navigate the design process with tight deadlines and limited funding available. We're really cutting down our features for now to create an MVP which we can then continually iterate on once we receive user feedback.
